# 코드리뷰 기준 (ClipNote iOS)

> 자동 CI(빌드) + 작성자(Claude)의 `/code-review`가 이 기준을 따른다. 리뷰 코멘트는 **한국어**로.

## 무엇을 "중요"로 볼지
아래만 **중요(반드시 지적)**. 나머지는 사소(nit) 이하:
1. **정확성/버그**: 로직 오류, 강제 언래핑(`!`)/강제 캐스팅으로 인한 크래시 가능성, 경계값, 옵셔널 처리 누락.
2. **메모리**: 클로저·델리게이트 순환참조(`[weak self]` 누락), 대용량 리소스 누수.
3. **동시성(Swift 6)**: 액터 격리 위반, 메인 스레드 밖 UI 갱신, `Sendable`/data race, `@MainActor` 누락.
4. **보안**: 시크릿·토큰 하드코딩(→ `Secrets.xcconfig`/Keychain), 민감정보 로그, API 응답 신뢰.
5. **리팩토링·단순화**: 중복 로직, 과한 뷰 복잡도, 재사용 가능 부분 — **제안(코멘트)** 으로만, 직접 커밋하지 않는다.

## 프로젝트 규칙
- XcodeGen: 타깃/설정 변경은 `project.yml`에서. `.xcodeproj` 직접 수정 금지(생성물).
- 시크릿은 `Secrets.xcconfig`. 커밋 금지.
- SwiftUI: 뷰 분리, 상태 남용·불필요 재렌더 주의, 접근성(Dynamic Type) 고려.

## 건너뛸 것 (지적하지 말 것)
- 생성물(`*.xcodeproj`), `*.md`.
- 개인 취향 수준의 스타일/네이밍.

## 항상 확인
- `xcodebuild build`가 깨질 컴파일 오류 없는지.
- 시크릿이 코드/커밋에 노출되지 않는지.
- 메인 액터/동시성 규칙을 지키는지.
